## Regarding `qf_auth_remote_data_source.dart`

The auth data source file you referenced implements the **Quran Foundation OAuth2 PKCE flow** visible in the video as the "Not signed in / Tap to sign in" prompt in the navigation drawer (timestamp ~1:20). Key capabilities of this data source that relate to the video:

- `login()` — triggers the AppAuth authorize+token exchange flow shown in the sign-in prompt
- `isAuthenticated()` — checks token expiry (the drawer shows "Not signed in" when no tokens exist)
- `revokeAndDeleteData()` — the "Delete My Data" feature mentioned in the changelog
- Token storage uses `flutter_secure_storage` for access/refresh/ID tokens
