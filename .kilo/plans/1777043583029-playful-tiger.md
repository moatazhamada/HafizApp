## Part 1: Add Quran MCP to both Claude Code and Kilo

### 1a. Claude Code — `.claude/settings.json` (CREATE)

Create `.claude/settings.json` with Quran MCP SSE server so Claude Code can call `fetch_quran`, `fetch_tafsir`, etc.

```json
{
  "mcpServers": {
    "quran-mcp": {
      "type": "sse",
      "url": "https://mcp.quran.ai/sse"
    }
  }
}
```

### 1b. Kilo — `.kilo/kilo.json` (CREATE)

Create `.kilo/kilo.json` with the same Quran MCP as a remote MCP server (Kilo uses `"type": "remote"` instead of `"type": "sse"`):

```jsonc
{
  "mcp": {
    "quran-mcp": {
      "type": "remote",
      "url": "https://mcp.quran.ai/sse",
      "enabled": true
    }
  }
}
```

## Part 4: Add QF Content API — Tafsir Integration

### 4a. New data source: `lib/data/datasource/tafsir/qf_tafsir_remote_data_source.dart` (CREATE)

- `QfTafsirRemoteDataSource` interface + impl
- `GET ${QfApiConfig.apiBaseUrl}/content/v1/tafsirs/{tafsirId}/by_ayah/{verseKey}`
- verseKey format: `"36:26"` (surah:ayah)
- Default tafsirId: `'en-tafsir-ibn-kathir'`

### 4b. Edit `lib/data/repository/tafsir/tafsir_repository_impl.dart`

- Inject `QfTafsirRemoteDataSource` alongside existing `TafsirRemoteDataSource`
- Try QF Content API first, fall back to existing Quran.com v4 source on failure

### 4c. Edit `lib/injection_container.dart`

- Register `QfTafsirRemoteDataSource` using the authenticated Dio instance

## Part 5: Verse Study Screen — Quran MCP Showcase Feature

### 5a. New data source: `lib/data/datasource/verse_study/qf_verse_study_remote_data_source.dart` (CREATE)

Three QF Content API calls per verse key (`surah:ayah`):
- `GET .../content/v1/quran/verses/by_key/{verse_key}` — Arabic text
- `GET .../content/v1/translations/{translation_id}/by_ayah/{verse_key}` — English translation
- `GET .../content/v1/tafsirs/{tafsir_id}/by_ayah/{verse_key}` — Tafsir

Returns a combined model with all three.

### 5b. New BLoC: `lib/presentation/verse_study/bloc/verse_study_bloc.dart` + event/state parts (CREATE)

- Event: `LoadVerseStudy(String verseKey)`
- States: `VerseStudyLoading`, `VerseStudyLoaded(arabicText, translation, tafsir)`, `VerseStudyError(message)`

### 5c. New screen: `lib/presentation/verse_study/verse_study_screen.dart` (CREATE)

- Shows: Arabic verse text (RTL) + English translation + Tafsir summary
- Receives `verseKey` as route argument
- Loading/error states via BLoC

### 5d. Edit `lib/routes/app_routes.dart`

- Add `verseStudyScreen = '/verse_study'` constant
- Add route builder accepting `verseKey` argument

### 5e. Edit `lib/presentation/surah_screen/surah_screen.dart`

- In `_showVerseMenu()` (~line 1135), add a "Study" `ListTile` after the Tafsir option
- Navigates to `/verse_study` with `{'verseKey': '${surah!.id}:${aya.verseNumber}'}`

### 5f. Edit `lib/injection_container.dart`

- Register `QfVerseStudyRemoteDataSource`, `VerseStudyBloc`
