## What You Need to Do (Priority Order)

### Phase 1: Qualify (MUST DO — no exceptions)

#### 1.1 Request OAuth2 Access from Quran Foundation
- Go to https://api-docs.quran.foundation/request-access
- Submit your app with redirect URI (e.g., `hafizapp://oauth/callback`)
- You need `client_id` and `client_secret` before any other work
- **Email developers@quran.com if it takes more than 2 days**

#### 1.2 Implement OAuth2 Authorization Code + PKCE (Android)
- Use `flutter_appauth` package (or AppAuth-Android for native)
- Implement: login screen → PKCE generation → redirect to QF login → callback → token exchange
- Store tokens securely (flutter_secure_storage)
- Implement token refresh
- Add user profile screen showing name from ID token
- **Key scopes needed**: `openid offline_access user bookmark collection reading_session goal streak`

#### 1.3 Integrate at Least One User API — Bookmarks (Best Fit)
You already have local bookmarks in Hive. Replace/augment with QF Bookmarks API:
- `GET /auth/v1/bookmarks` — fetch user's Quran.com bookmarks
- `POST /auth/v1/bookmarks` — create bookmark (sync local → remote)
- `DELETE /auth/v1/bookmarks/{id}` — delete bookmark
- **Sync strategy**: Bidirectional sync between local Hive bookmarks and QF API bookmarks

### Phase 2: Be Competitive (IMPORTANT — differentiates from other entries)

#### 2.1 Sync Khatmah/Reading Progress with QF Reading Sessions API
- You have a khatmah tracker (daily reading goals, streak counter, weekly heatmap)
- Map this to QF Reading Session API: `GET/POST /auth/v1/reading-sessions`
- This replaces Firebase Firestore cloud sync with the QF ecosystem — much more aligned with hackathon goals

#### 2.2 Sync Streaks with QF Streak API
- Your khatmah streak counter → QF Streak Tracking API
- `GET /auth/v1/streaks` — show Quran.com streak in your app
- `POST /auth/v1/streaks` — update streak from your app's reading activity

#### 2.3 Sync Memorization Goals with QF Goals API
- Your memorization tracker → QF Goals API
- `GET/POST /auth/v1/goals` — reading/memorization goals
- This is unique — showing memorization progress as goals

#### 2.4 Collections Integration
- Allow users to save verses to QF Collections (e.g., "Verses I'm Memorizing", "Difficult Verses")
- `GET/POST /auth/v1/collections` — CRUD collections
- This directly ties into your memorization workflow

### Phase 3: Win (DIFFERENTIATORS — push toward 1st place)

#### 3.1 Cross-App Sync Story
- **This is your strongest narrative**: "A hafiz's memorization data syncs across Quran.com, the HafizApp, and any other QF Connected App"
- When a user bookmarks a verse on Quran.com, it appears in HafizApp
- When they mark a verse memorized in HafizApp, it's reflected in their QF profile
- This ecosystem alignment is exactly what the email mentioned for "Connected Apps ecosystem consideration"

#### 3.2 Replace Firebase Cloud Sync with QF APIs
- Currently cloud sync uses anonymous Firebase auth + Firestore
- Replace with QF User APIs for bookmarks, reading sessions, goals, streaks
- Keep Firebase for Crashlytics/Analytics (those are fine)
- This shows deep commitment to the QF ecosystem

#### 3.3 Demo Video (2-3 minutes) — Critical for judging
Structure:
1. **Hook (15s)**: "What if your Quran memorization progress followed you everywhere?"
2. **Problem (30s)**: Memorization data is trapped in one device/app
3. **Solution (60s)**: Show OAuth login → sync bookmarks from Quran.com → voice verification → memorization tracking → all synced to QF
4. **Technical (30s)**: Show architecture diagram, API integrations
5. **Close (15s)**: Call to action, team info

#### 3.4 Submission Package
- GitHub repo (clean, well-documented)
- Working APK (or Play Store link)
- Demo video (upload to YouTube, unlisted)
- API usage documentation
- Architecture diagram showing QF API integration

### Backend Component (for confidential client)
Since QF issues confidential clients by default, you need a minimal backend for token exchange:
- Option A: Simple Node.js/Express server (one endpoint: `/api/auth/qf/exchange`)
- Option B: Firebase Cloud Functions (already using Firebase)
- Option C: Request public client from QF (simpler but may not be granted)

**Recommended**: Firebase Cloud Functions — you're already on Firebase, so adding one function for token exchange is minimal effort.
